# React Accessibility Integration Summary

## Quick Overview

The React accessibility integration adds comprehensive accessibility tools and components to React applications generated by PTG UI Schematics.

## Usage in Generated Project

### 1. Initialize Axe (main.tsx)
```typescript
import { initializeAxe } from './app/accessibility/utils/axe-helper';

if (process.env.NODE_ENV !== 'production') {
  initializeAxe(React, ReactDOM, 1000);
}
```

### 2. Use Components
```typescript
import { SkipLink, LiveRegion } from './app/accessibility/components/A11yComponents';
```

### 3. Use Utilities
```typescript
import { announceToScreenReader } from './app/accessibility/utils/a11y-utils';
```

## Features

### Automated Testing
- Axe-core runs automatically in dev mode
- Reports WCAG violations to console
- Supports custom rule configurations

### Accessible Components
- **SkipLink**: Skip navigation for keyboard users
- **LiveRegion**: Announce dynamic content changes
- **VisuallyHidden**: Hide content visually, not from screen readers
- **FocusTrap**: Trap focus within modals/dialogs
- **A11yAnnouncer**: Global announcer with React portals

### Utility Functions
- `announceToScreenReader()`: Announce messages
- `trapFocus()`: Manual focus trapping
- `generateA11yId()`: Generate unique IDs
- `prefersReducedMotion()`: Check motion preference
- `skipToMainContent()`: Skip navigation programmatically

### ESLint Rules
- Alt text validation
- Anchor validity checks
- ARIA role validation
- Click event keyboard equivalents
- Label-control associations

## Common Issues and Solutions

### Issue: Axe not running
**Cause**: Not initialized or not in dev mode
**Fix**: Add initializeAxe() in main.tsx before render

### Issue: ESLint jsx-a11y rules not working
**Cause**: Plugin not in config or dependencies
**Fix**: Verify eslint.config.js has 'jsx-a11y': jsxA11y in plugins

### Issue: Components not found
**Cause**: Import path incorrect
**Fix**: Use `./app/accessibility/components/A11yComponents`

### Issue: TypeScript errors in utilities
**Cause**: Type definitions missing
**Fix**: Ensure @types/react and @types/react-dom are installed
