# ESLint Fixes for Angular Schematics

## Overview
This document outlines all the ESLint fixes applied to the Angular application templates generated by the PTG UI Schematics.

## Issues Fixed

### 1. Line Endings (CRLF → LF)
**Problem**: Template files used Windows line endings (CRLF) which caused ESLint errors on Unix systems.

**Files Fixed**:
- `app.component.ts.template`
- `core.module.ts.template`
- `shared.module.ts.template`
- `main.ts.template`
- `app.component.html.template`
- `i18n-files/app.component.ts.template`
- `i18n-files/app.component.html.template`

**Solution**: Converted all line endings to LF format for cross-platform compatibility.

### 2. ESLint Configuration Improvements

#### Disabled Conflicting Rules
- `@typescript-eslint/indent`: Disabled to avoid conflicts with Prettier
- `@typescript-eslint/comma-dangle`: Disabled to let Prettier handle it

#### Enhanced Prettier Configuration
Added comprehensive Prettier settings:
```json
{
  "endOfLine": "auto",
  "singleQuote": true,
  "trailingComma": "es5",
  "tabWidth": 2,
  "semi": true,
  "printWidth": 80
```

#### Improved no-unused-vars Rule
Enhanced the rule to be more flexible:
```json
{
  "argsIgnorePattern": "^_",
  "varsIgnorePattern": "^_",
  "ignoreRestSiblings": true,
  "destructuredArrayIgnorePattern": "^_"

```

### 3. Angular-Specific Rule Adjustments

#### Made Rules More Lenient for Generated Code
- `@angular-eslint/no-inputs-metadata-property`: Changed from "warn" to "off"
- `@angular-eslint/no-outputs-metadata-property`: Changed from "warn" to "off"
- Added `@angular-eslint/component-class-suffix`: "error"
- Added `@angular-eslint/directive-class-suffix`: "error"
- Added `@angular-eslint/pipe-prefix`: "off"

## Usage Instructions

### For Generated Applications

1. **Run linting check**:
   ```bash
   npm run lint
   ```

2. **Fix all auto-fixable issues**:
   ```bash
   npm run fix-all
   ```

3. **Check for CI/CD**:
   ```bash
   npm run lint:ci
   ```

4. **Format code**:
   ```bash
   npm run format
   ```

### Common ESLint Issues and Solutions

#### Issue: "Expected linebreaks to be 'LF' but found 'CRLF'"
**Solution**: Run `npm run format` or configure your editor to use LF line endings.

#### Issue: "Missing trailing comma"
**Solution**: Run `npm run format` - Prettier will handle this automatically.

#### Issue: "Unexpected console statement"
**Solution**: Use `console.warn()` or `console.error()` instead of `console.log()`, or disable the rule for specific lines with `// eslint-disable-next-line no-console`.

#### Issue: "Import not found" or "Unable to resolve path"
**Solution**: This is usually handled by the `import/no-unresolved: "off"` rule in the configuration.

## Troubleshooting

### If you encounter persistent ESLint errors:

1. **Clear ESLint cache**:
   ```bash
   npx eslint --cache-location .eslintcache --cache-strategy content src/**/*.{ts,html} --fix
   ```

2. **Regenerate node_modules**:
   ```bash
   rm -rf node_modules package-lock.json
   npm install
   ```

3. **Check your IDE ESLint extension settings** to ensure it's using the project configuration.
